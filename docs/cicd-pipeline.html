CI/CD Pipeline — Task Manager

GitFlow on GitHub Actions · two pipelines (App CI → Infra CD) chained via workflow_run

GitHub Actions workflows: 2
trigger · push
feature/*
Any push to a feature branch
→ App CI runs · no deploy
trigger · push / merge
develop
Merge of feature branch into develop
→ App CI + auto-deploy to dev
trigger · push / merge
main
Merge of develop into main
→ App CI + gated deploy to prod
① App CI .github/workflows/app-ci.yml
runs-on: ubuntu-latest · on: push (all branches)
STEP 1
Lint
golangci-lint run ./...
STEP 2
Unit tests
go test -race ./...
STEP 3 · only on develop / main
WIF auth + Buildx
google-github-actions/auth · docker/setup-buildx-action
STEP 4 · only on develop / main
Build & Push to Artifact Registry
docker/build-push-action · tags: :<sha> + :latest · GHA cache
◆ if branch = develop → push to
europe-west9-docker.pkg.dev/.../task-manager-dev :<commit-sha> :latest
◆ if branch = main → push to
europe-west9-docker.pkg.dev/.../task-manager-prod :<commit-sha> :latest
on App CI success (develop / main)
trigger: workflow_run
② Infra CD .github/workflows/infra-cd.yml
runs-on: arc-runner-set (self-hosted, ARC on GKE) · terraform 1.9.8 · on: workflow_run (App CI completed)
